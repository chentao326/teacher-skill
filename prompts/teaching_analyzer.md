# 教学风格分析 Prompt

## 任务

你将收到 **{name}** 的原材料（课程字幕、视频文字稿、教学文章、课堂录音转写等）。
从中提取他的教学风格与方法，用于构建教学风格档案。

**核心原则：只提取教学相关内容，不推断，有依据才写，没有就标注"原材料不足"。**

---

## 原材料预处理

### 1. 素材清洗
- 去除时间戳、字幕编号、SRT/VTT 格式标记等非内容信息
- 保留完整的对话和讲解内容
- 标注素材来源（如"第3节课"、"2024年公开课"）

### 2. 原材料分类与权重

对素材中的每段内容进行分类，赋予不同权重：

| 内容类型 | 判断标准 | 权重 | 说明 |
|---------|---------|------|------|
| **长段讲解** | 单次连续发言 > 100 字，涉及概念解释、知识推导 | 高 | 教学风格的核心证据 |
| **互动对话** | 师生问答、课堂讨论、对学生的反馈回应 | 中 | 反映互动模式和反馈风格 |
| **日常寒暄** | 开场白、闲聊、课程通知、行政事项 | 低 | 仅用于提取表达习惯，不作为教学风格依据 |
| **板书/PPT 文字** | 板书内容、PPT 上的要点文字 | 中 | 反映内容组织方式和重点标记习惯 |
| **作业/考试要求** | 布置作业、说明考试要求 | 中 | 反映评估方法和教学目标 |

## 通用提取维度（所有学科适用）

### 1. 教学目标设定

从原材料中识别：
- 教师对学习目标的表述方式（明确列出 vs 融入讲解中）
- 对"学会什么"的定义（知识记忆 / 理解应用 / 分析创造）
- 是否区分不同学生的目标（分层目标）
- 课程/章节开头的目标说明方式

```
输出格式：
目标表述方式：[描述]
目标层次：[记忆/理解/应用/分析/创造]
分层目标：[有/无，描述]
```

### 2. 内容组织方式

从讲解结构中提取：
- 整体结构偏好（总分总 / 线性递进 / 问题驱动 / 案例驱动 / 知识图谱式）
- 知识点之间的衔接方式（过渡语、逻辑链、类比桥梁）
- 重点内容的标记方式（"注意"/"关键"/"记住"/板书加粗/重复强调）
- 难点内容的处理方式（放慢节奏/增加类比/分步拆解/先跳过后续回顾）
- 每个知识块的讲解密度（一次讲多少内容再停下来）

```
输出格式：
结构偏好：[描述]
衔接方式：[描述]
重点标记：[方式 + 示例]
难点处理：[描述]
讲解密度：[描述]
```

### 3. 评估方法

从原材料中识别：
- 理解检查的方式（提问/练习/观察/自评）
- 检查的频率（每概念/每小节/每节课）
- 对学生错误的反馈方式（直接纠正/引导发现/忽略/延迟处理）
- 练习/作业的设计特点（题型/难度梯度/数量）
- 是否使用评分/等级反馈

```
输出格式：
检查方式：[描述]
检查频率：[描述]
错误反馈：[描述]
练习特点：[描述]
```

### 4. 学生互动模式

从师生对话中提取：
- 提问类型（开放式/封闭式/苏格拉底式/选择式/假设式）
- 提问频率和节奏
- 等待学生回答的耐心程度（给多少时间思考）
- 对正确回答的反馈（简短肯定/详细表扬/追问延伸）
- 对错误回答的反馈（直接纠正/引导纠正/换角度提示/鼓励再试）
- 对"不知道"的处理方式（给提示/换简单问题/直接讲解/让其他学生回答）
- 称呼学生的方式

```
输出格式：
提问类型：[描述]
提问频率：[描述]
答对反馈：[描述 + 原话示例]
答错反馈：[描述 + 原话示例]
"不知道"处理：[描述]
称呼方式：[描述]
```

## 学科专项提取

根据 {name} 的学科，参考 `subjects/` 下的学科模板，重点提取对应维度：

### 数学类（数学/统计/概率等）

**概念引入方式**：
- 从直觉/几何直观出发 vs 从形式化定义出发
- 是否先给具体例子再抽象
- 对公式的处理（先推导再记 vs 先记住再用）

**证明与推导**：
- 证明的详细程度（完整写出/关键步骤/跳步）
- 是否要求学生自己证明
- 对"为什么"的重视程度

**常见数学教学误区识别**：
- 是否指出学生常见错误
- 对计算准确性的要求

### 编程类（计算机科学/软件工程/编程语言等）

**代码演示习惯**：
- 是否边讲边写代码
- 代码的注释风格和详细程度
- 是否先讲思路再写代码 vs 直接写代码边写边解释
- 错误演示（是否故意写 bug 让学生找）

**概念讲解方式**：
- 用伪代码/流程图/实际代码的比例
- 对底层原理的讲解深度
- API/工具的使用教学方式

**实践环节设计**：
- 课后作业/练习的形式
- 对调试能力的教学
- 项目/案例的规模和复杂度

### 理科类（物理/化学/生物等）

**实验与演示**：
- 是否引用实验现象
- 用生活中的物理/化学现象做类比的频率
- 对公式推导 vs 概念理解的侧重

**可视化使用**：
- 图表/图像/动画的使用频率
- 板书画图的习惯

### 文科类（历史/文学/哲学/语言等）

**文本分析方式**：
- 是否逐段/逐句分析
- 引用原文的频率
- 对不同观点的呈现方式

**讨论与思辨**：
- 是否鼓励学生发表观点
- 对争议性话题的处理方式
- 批判性思维的培养方式

### 语言类（英语/外语/语文等）

**语言输入方式**：
- 目标语言的使用比例
- 对语法/词汇/发音的侧重
- 是否使用沉浸式教学

**练习设计**：
- 口语练习的形式和频率
- 写作练习的反馈方式
- 纠错策略（即时纠错/延迟纠错/选择性纠错）

---

## 输出要求

- 语言：中文
- 没有信息的维度：标注 `（原材料不足，建议追加相关素材）`
- 有原文依据的结论：加引号标注原话
- 输出结果直接用于生成教学风格档案（`teaching_builder.md`），要求具体可执行
- 不要写"可能""倾向于"这类模糊表述，要么有依据直接写，要么标注不足
- 区分"高置信度结论"（多条原材料支撑）和"低置信度推测"（仅1条支撑），后者需标注
