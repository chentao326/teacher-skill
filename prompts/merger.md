# 增量 Merge Prompt

## 任务

你将收到：
1. 现有的教学风格档案内容（由 `teaching_builder.md` 生成的档案文件）
2. 新的原材料内容（新的课程字幕、视频文字稿、教学文本等）

你的任务是判断新内容应该更新档案的哪个部分，并输出增量更新内容。

**核心原则：只追加增量，不覆盖已有结论。如有冲突，输出冲突提示让用户决定。**

---

## Step 1：分类判断

将新内容中的每条信息归类到档案的对应层级：

| 信息类型 | 归入层级 |
|---------|---------|
| 教学信念、教学原则、核心方法论 | -> Layer 0 |
| 身份信息（学科/机构/教龄等） | -> Layer 1 |
| 口头禅、高频词、说话方式、对话示例 | -> Layer 2 |
| 教学决策、优先级、推进/回避触发、错误处理 | -> Layer 3 |
| 对不同水平学生的互动行为 | -> Layer 4 |
| 教学禁忌、回避话题、不喜欢的方式 | -> Layer 5 |
| 两者都有 | -> 分别归入对应层级 |

---

## Step 2：检查冲突

对比新内容与现有档案内容：

- 如果新内容**补充**了现有信息（增加了新细节、新场景、新示例）-> 直接追加
- 如果新内容**确认**了现有信息（说了同样的事）-> 忽略（不重复写）
- 如果新内容**细化**了现有信息（更具体的描述、更准确的措辞）-> 用更具体的版本替换，标注更新
- 如果新内容**与现有信息矛盾** -> 输出冲突提示：

```
发现冲突：
- 现有：{现有描述}
- 新发现：{新内容描述}
- 来源：{素材名称/时间}
建议：[保留现有 / 更新为新内容 / 两者都保留并标注来源]
请用户决定。
```

---

## Step 3：生成更新 Patch

对每个需要更新的层级，输出格式：

```
=== Layer 0 更新 ===
[追加到"核心教学信念"节]
- {新行为规则}

=== Layer 2 更新 ===
[追加到"口头禅与高频词"节]
- 新口头禅："{xxx}"
[追加到"你会怎么说"节]
> {新场景}：
> 你：{新对话示例}

=== Layer 3 更新 ===
[追加到"你会推进的情况"节]
- {新触发条件}

=== Layer 4 更新 ===
[追加到"对弱基础学生"节]
- {新互动行为描述}

=== Layer 5 更新 ===
[追加到"你不喜欢的教学方式"节]
- {新事项}

[无更新] 或 [以上层级有更新]
```

---

## Step 4：生成更新摘要

向用户展示：

```
本次更新摘要：
- Layer 0：追加了 {N} 条新规则（{简要描述}）
- Layer 1：更新了 {N} 项身份信息（{简要描述}）
- Layer 2：追加了 {N} 条表达特征（{简要描述}）
- Layer 3：追加了 {N} 条教学决策（{简要描述}）
- Layer 4：追加了 {N} 条互动行为（{简要描述}）
- Layer 5：追加了 {N} 条边界规则（{简要描述}）
- 发现 {N} 处冲突，需要你确认（见上方）
档案版本将从 v{N} 升级到 v{N+1}。
确认应用更新？
```

## 特殊处理规则

### 新素材与标签冲突

如果新素材中的发现与用户最初填写的教学标签（来自 `intake.md`）矛盾：

```
发现标签冲突：
- 原始标签：{标签名}
- 素材证据：{具体描述 + 原话引用}
建议：[更新标签 / 保留标签但标注例外 / 需要更多素材确认]
```

### 置信度提升

当新素材为已有结论提供了额外支撑时：
- 原本标注"中等置信度"的结论，如果新素材再次确认，升级为高置信度（去掉标注）
- 原本标注"基于标签推断"的结论，如果有实际素材支撑，更新为有依据的结论

### 对话示例更新

Layer 2 的对话示例可以持续丰富：
- 新增不同场景下的对话示例
- 如果新素材中有更典型的对话，可以替换现有示例（标注"替换了更典型的示例"）
