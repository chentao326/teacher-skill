# 出题模板库

## 使用说明

根据用户的能力等级和当前教学阶段，选择合适的题型和模板。
题目必须**基于已讲授的内容**，不要考查未教过的知识。

---

## 一、理解检查题（每个概念讲完后使用）

### 1.1 概念理解类

**弱基础 — 选择题模板**：
```
关于 [概念]，下面哪个说法是正确的？
A. [错误说法1 - 常见误解]
B. [正确说法]
C. [错误说法2 - 混淆概念]
D. [错误说法3 - 相反说法]
```

**中等基础 — 复述题模板**：
```
请用自己的话解释一下：[概念] 是什么？它解决了什么问题？
```

**强基础 — 比较分析题模板**：
```
[概念A] 和 [概念B] 有什么区别？分别在什么场景下使用？
```

### 1.2 原理理解类

**弱基础 — 判断题模板**：
```
判断以下说法是否正确：
"[关于概念的陈述]"
为什么？
```

**中等基础 — 填空题模板**：
```
[概念] 的核心步骤是：
1. [步骤1]
2. [步骤2]
3. _____（请补充）
```

**强基础 — 原理推导题模板**：
```
为什么 [概念] 要这样设计？如果不这样设计，会有什么问题？
```

## 二、应用练习题（每个学习单元结束后使用）

### 2.1 场景应用类

**弱基础 — 选择应用题模板**：
```
在以下哪个场景中，最适合使用 [概念]？
A. [不相关场景]
B. [相关场景]
C. [部分相关但不最适合的场景]
D. [相反场景]
```

**中等基础 — 简单应用题模板**：
```
假设你遇到了这样的问题：[场景描述]
你会怎么用 [概念] 来解决？请描述你的思路。
提示：可以按以下步骤思考：
1. 先分析 [关键因素]
2. 然后选择 [方法]
3. 最后 [执行步骤]
```

**强基础 — 综合应用题模板**：
```
[复杂场景描述，涉及多个概念]

请设计一个解决方案，需要用到：
- [概念A]
- [概念B]
- [概念C]

要求：
1. 说明你的整体思路
2. 解释每个概念在这个方案中的作用
3. 讨论可能的替代方案
```

### 2.2 实操类（适用于编程/技术类内容）

**弱基础 — 代码填空模板**：
```
请补全以下代码，实现 [功能]：

def [函数名](参数):
    # 第一步：[描述]
    _____
    # 第二步：[描述]
    _____
    # 第三步：返回结果
    return _____
```

**中等基础 — 代码改错模板**：
```
以下代码试图实现 [功能]，但有 [N] 个错误。请找出并修正：

[有错误的代码]

提示：注意 [易错点]。
```

**强基础 — 设计实现模板**：
```
请从零开始实现 [功能]，要求：
1. 使用 [概念/模式]
2. 考虑 [边界情况]
3. 代码清晰、有注释
```

## 三、阶段性回顾题（每3-5个单元后使用）

### 3.1 知识串联题

```
我们已经学了 [概念A]、[概念B]、[概念C]。
请描述它们之间的关系：它们是如何协同工作的？
```

### 3.2 综合应用题

```
[综合场景，需要运用多个已学概念]

请用你学过的知识来解决，并在回答中标注你用到了哪些概念。
```

### 3.3 概念地图题

```
请把以下概念按照它们的关系排列/连线：
[概念列表]

提示：想想哪些是基础概念，哪些是建立在其他概念之上的。
```

## 四、出题原则

### 必须遵守
- ✅ 题目基于已讲授内容，不超纲
- ✅ 错误选项要有迷惑性（常见误解），不能太明显
- ✅ 每道题只考查一个核心知识点
- ✅ 题目语言要简洁清晰
- ✅ 应用题的场景要贴近实际

### 绝对禁止
- ❌ 出陷阱题或故意误导的题目
- ❌ 一次出太多题（最多 2-3 道）
- ❌ 题目中使用未解释的新术语
- ❌ 出过于刁钻的边界情况（除非是强基础用户）
- ❌ 题目措辞含糊不清

### 难度控制
| 能力等级 | 理解检查 | 应用练习 | 阶段回顾 |
|---------|---------|---------|---------|
| 弱 | 选择/判断 | 选择应用 | 简单串联 |
| 中 | 复述/填空 | 引导应用 | 综合应用 |
| 强 | 分析/比较 | 综合设计 | 深度讨论 |
